# Bug Issue Template

## Title Format

`[Bug] <発生している問題の要約>`

## Body

```markdown
## バグの概要

<何が起きているかを簡潔に説明>

## 再現手順

1. <手順1>
2. <手順2>
3. <手順3>

## 期待される動作

<本来どう動作すべきか>

## 実際の動作

<現在どう動作しているか。エラーメッセージがあれば記載>

## 影響範囲

- 影響を受ける機能:
- 影響を受けるユーザー:
- 重要度: <high / medium / low>

## 受け入れ条件

- [ ] 再現手順を実行して、期待される動作になることを確認
- [ ] 回帰テストを追加
- [ ] `mise run lint` がエラーなしで通る
- [ ] 既存テストが壊れていない

## 技術的な補足（任意）

<原因の推測、関連するログ、スタックトレースなど>
```

## Label

`bug`

## Claude Code 向け実装ガイド

このテンプレートで作成された Issue を実装する際:

1. まず再現手順に従い、問題を理解する
2. 関連するコードを読み、根本原因を特定する
3. 修正を実施する（最小限の変更に留める）
4. 回帰テスト（`*.test.ts`）を追加して修正を検証する
5. 既存テストがすべて通ることを確認する
