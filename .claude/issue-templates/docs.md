# Docs Issue Template

## Title Format

`[Docs] <ドキュメント変更の要約>`

## Body

```markdown
## 概要

<どのドキュメントを、なぜ追加・更新するか>

## 対象ドキュメント

- <対象ファイルパスまたは新規作成先>

## 内容

<記載すべき内容の概要。箇条書きで構成を示す>

- <セクション1>
- <セクション2>
- <セクション3>

## 受け入れ条件

- [ ] <ドキュメントの完成条件1>
- [ ] <ドキュメントの完成条件2>
- [ ] 既存ドキュメントとの整合性が取れている

## 技術的な補足（任意）

<参照すべき既存ドキュメント、関連 Issue など>
```

## Label

`docs`

## Claude Code 向け実装ガイド

このテンプレートで作成された Issue を実装する際:

1. 対象ドキュメントの既存内容を確認する
2. 関連するコードや設定を読み、正確な情報を記載する
3. 既存ドキュメントのフォーマット・文体に合わせる
4. コードサンプルを含む場合は実際に動作するコードを記載する
