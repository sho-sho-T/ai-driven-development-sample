# Task Issue Template

## Title Format

`[Task] <実行するタスクの要約>`

## Body

```markdown
## 概要

<何を行うタスクか。技術的な背景を含めて簡潔に説明>

## 目的

<このタスクを行う理由。どのような技術的課題を解決するか>

## 作業内容

1. <具体的な作業ステップ1>
2. <具体的な作業ステップ2>
3. <具体的な作業ステップ3>

## 受け入れ条件

- [ ] <完了を判定できる具体的な条件1>
- [ ] <完了を判定できる具体的な条件2>
- [ ] <完了を判定できる具体的な条件3>
- [ ] `mise run lint` がエラーなしで通る
- [ ] 既存テストが壊れていない

## 対象ファイル・モジュール

- <変更対象のファイルパスまたはモジュール名>

## 技術的な補足

<設定値、参照ドキュメント、依存関係など>
```

## Label

`task`

## Claude Code 向け実装ガイド

このテンプレートで作成された Issue を実装する際:

1. 作業内容を上から順に実施する
2. 対象ファイル・モジュールを先に読み込み、既存コードのパターンを把握する
3. モジュール境界（contracts 経由）を遵守する
4. リファクタリングの場合は既存テストが通ることを確認しながら進める
5. 設定変更の場合は影響範囲を確認する
