# Enhancement Issue Template

## Title Format

`[Enhancement] <改善内容の要約>`

## Body

```markdown
## 概要

<どの既存機能を、どのように改善するか>

## 現状の課題

<現在の実装の問題点。パフォーマンス、UX、保守性などの観点>

## 改善内容

<具体的に何をどう変えるか>

## 受け入れ条件

- [ ] <改善が達成されたことを検証できる条件1>
- [ ] <改善が達成されたことを検証できる条件2>
- [ ] <改善が達成されたことを検証できる条件3>
- [ ] `mise run lint` がエラーなしで通る
- [ ] 既存テストが壊れていない

## 対象ファイル・モジュール

- <変更対象のファイルパスまたはモジュール名>

## 技術的な補足（任意）

<ベンチマーク結果、参考実装、関連 Issue など>
```

## Label

`enhancement`

## Claude Code 向け実装ガイド

このテンプレートで作成された Issue を実装する際:

1. 現状の課題を理解するため、対象コードを読み込む
2. 改善内容に基づいて変更を計画する
3. 既存の振る舞いを壊さないよう、段階的に変更する
4. パフォーマンス改善の場合は改善前後の比較ができるようにする
5. 既存テストの更新が必要な場合は合わせて修正する
