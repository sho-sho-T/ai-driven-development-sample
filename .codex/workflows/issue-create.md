# issue-create ワークフロー

## トリガー

ユーザーが「Issue を作って」「〜の Issue を作成して」「〜を実装したい」と指示したとき。

## Flow

### 1. ユーザーの指示を解析

- タイトル（簡潔な1行）を生成
- 概要と受け入れ条件を抽出・整理

### 2. ラベルを自動判定

指示内容から適切なラベルを1つ選択する。

| 指示の特徴 | ラベル |
|---|---|
| ユーザー視点の機能要件、「〜できるようにする」 | `user-story` |
| 内部的な技術タスク、リファクタリング、設定変更 | `task` |
| ドキュメントの追加・更新 | `docs` |
| 不具合、エラー、期待と異なる動作 | `bug` |
| 既存機能の改善、パフォーマンス改善 | `enhancement` |

### 3. ラベルの存在確認

選択したラベルが存在しない場合は作成する。

```bash
gh label create "<label>" --force
```

### 4. Issue を作成

```bash
gh issue create \
  --title "<title>" \
  --body "## 概要
<ユーザーの指示から抽出した概要>

## 受け入れ条件
- [ ] <条件1>
- [ ] <条件2>
..." \
  --label "<label>"
```

### 5. Issue URL を出力

作成した Issue の URL を表示する。
